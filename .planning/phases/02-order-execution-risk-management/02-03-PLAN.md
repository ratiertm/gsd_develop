  <action>
Create `kiwoom_trader/core/position_tracker.py`:

Class `PositionTracker`:
- `__init__(self, risk_config: RiskConfig)`: stores risk_config, creates dict[str, Position] for _positions, float _daily_realized_pnl = 0.0
- `add_position(self, code, qty, avg_price) -> Position`: Create Position, set high_water_mark = avg_price, compute stop_loss/take_profit/trailing_stop from risk_config, store in _positions.
- `update_position(self, code, qty, avg_price)`: Update existing position. If qty == 0, remove position. Otherwise update qty and avg_price.
- `update_from_chejan(self, code, holding_qty, buy_price, current_price)`: If holding_qty > 0: add/update position. If holding_qty == 0: remove position. Compute unrealized_pnl = (current_price - buy_price) * holding_qty.
- `update_price(self, code, current_price)`: Update unrealized_pnl for position. Used on every real-time tick.
- `get_position(self, code) -> Position | None`
- `get_all_positions(self) -> dict[str, Position]`
- `get_unrealized_pnl(self) -> int`: Sum of all positions' unrealized_pnl.
- `get_daily_pnl(self) -> float`: _daily_realized_pnl + get_unrealized_pnl(). This is the key metric for daily loss limit (per RESEARCH.md Pitfall 4 -- include both realized AND unrealized).
- `get_total_invested(self) -> int`: Sum of (avg_price * qty) for all positions.
- `check_symbol_weight(self, code, order_amount, total_capital) -> bool`: Returns True if OK. Current position value for code + order_amount <= total_capital * max_symbol_weight_pct / 100.
- `check_max_positions(self, code) -> bool`: Returns True if OK. If code already in _positions, always True. Otherwise, len(_positions) < max_positions.
- `record_realized_pnl(self, amount: int)`: _daily_realized_pnl += amount.
- `reset_daily(self)`: _daily_realized_pnl = 0.0, clears for new trading day.

Write tests first (RED), then implement (GREEN). Use mock_risk_config fixture from conftest.py.

Create `kiwoom_trader/core/market_hours.py`:

Class `MarketHoursManager`:
- `__init__(self, risk_config: RiskConfig, time_func=None)`: stores risk_config. `time_func` is a callable returning datetime.time (default: `lambda: datetime.now().time()`). Parse time strings from risk_config into datetime.time objects for comparison.
- `get_market_state(self) -> MarketState`: Determines current state by comparing current time (via time_func) against configured boundaries:
  - Before auction_start_am (08:30): CLOSED
  - auction_start_am to auction_end_am (08:30-09:00): PRE_MARKET_AUCTION
  - auction_end_am to trading_start (09:00-09:05): MARKET_OPEN_BUFFER
  - trading_start to trading_end_new_buy (09:05-15:15): TRADING
  - trading_end_new_buy to auction_start_pm (15:15-15:20): CLOSING
  - auction_start_pm to auction_end_pm (15:20-15:30): CLOSING_AUCTION
  - After auction_end_pm (15:30+): CLOSED

  Use datetime.time comparison operators (NOT string comparison -- per RESEARCH.md "Don't Hand-Roll" section).

- `is_trading_allowed(self) -> bool`: get_market_state() == TRADING
- `is_new_buy_allowed(self) -> bool`: get_market_state() == TRADING (not CLOSING -- in CLOSING state, only sell/liquidation allowed)
- `is_order_blocked(self) -> bool`: state in {PRE_MARKET_AUCTION, CLOSING_AUCTION, MARKET_OPEN_BUFFER, CLOSED}
- `is_closing_time(self) -> bool`: get_market_state() == CLOSING

All time comparisons use datetime.time objects parsed from RiskConfig strings via `datetime.strptime(time_str, "%H:%M").time()`.

The `time_func` injection pattern enables deterministic testing without mocking datetime.

Write tests first (RED) using `time_func=lambda: datetime.time(hour, minute)` to inject specific times. Then implement (GREEN).
