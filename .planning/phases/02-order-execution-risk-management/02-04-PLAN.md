Create `kiwoom_trader/core/risk_manager.py`:

Use PyQt5 import fallback pattern (try/except with MagicMock) for cross-platform testing.

Class `RiskManager` (inherits QObject if PyQt5 available):
- Signals: `trigger_stop_loss(str, int)` (code, price), `trigger_take_profit(str, int)` (code, price), `trigger_trailing_stop(str, int)` (code, price), `daily_loss_limit_hit()`, `position_liquidated(str)` (code)
- `__init__(self, order_manager, position_tracker, market_hours, risk_config, total_capital: int)`:
  - Stores all dependencies. _daily_buy_blocked = False. total_capital is the account total for position limit calculations.

- `validate_order(self, code, side, qty, price) -> tuple[bool, str]`:
  Pre-trade validation gate (from RESEARCH.md Pattern 2). All checks must pass:
  1. market_hours.is_order_blocked() -> reject "Orders blocked during current market state"
  2. If side == BUY and not market_hours.is_new_buy_allowed() -> reject "New buys not allowed in current state"
  3. If side == BUY and _daily_buy_blocked -> reject "Daily loss limit reached, buys blocked"
  4. Daily P&L check: position_tracker.get_daily_pnl() / total_capital * 100 <= -daily_loss_limit_pct -> reject
  5. Symbol weight: position_tracker.check_symbol_weight(code, qty * price, total_capital) -> reject if False
  6. Max positions: position_tracker.check_max_positions(code) -> reject if False
  Return (True, "OK") if all pass.

- `on_price_update(self, code, data_dict)`:
  Called as RealDataManager subscriber. This is the real-time risk trigger loop (RESEARCH.md Pattern 3).
  1. Extract current_price = abs(int(data_dict.get(FID.CURRENT_PRICE, "0") or "0"))
  2. If current_price == 0: return
  3. Update position_tracker.update_price(code, current_price)
  4. Get position = position_tracker.get_position(code). If None: return.
  5. **Daily loss check** (per user decision: realized + unrealized, RESEARCH.md Pitfall 4):
     daily_pnl_pct = position_tracker.get_daily_pnl() / total_capital * 100
     If daily_pnl_pct <= -risk_config.daily_loss_limit_pct:
       -> emit daily_loss_limit_hit, call liquidate_all(), set _daily_buy_blocked = True, return
  6. **Stop-loss** check: if current_price <= position.stop_loss_price:
     -> emit trigger_stop_loss, submit market sell via order_manager
  7. **Take-profit** check: if current_price >= position.take_profit_price:
     -> emit trigger_take_profit, submit market sell via order_manager
  8. **Trailing stop** update + check:
     If current_price > position.high_water_mark:
       position.high_water_mark = current_price
       position.trailing_stop_price = int(current_price * (1 - risk_config.trailing_stop_pct / 100))
     elif current_price <= position.trailing_stop_price and position.trailing_stop_price > 0:
       -> emit trigger_trailing_stop, submit market sell via order_manager

- `liquidate_all(self)`:
  For each position in position_tracker.get_all_positions(), sorted by unrealized_pnl ascending (worst first, per RESEARCH.md Open Question 3):
  Submit market sell order via order_manager.submit_order(code, SELL, qty, 0, MARKET).
  Set _daily_buy_blocked = True.

- `on_closing_time(self)`:
  Called when MarketHoursManager transitions to CLOSING state.
  Submit market sell for all held positions (day-trading rule: no overnight positions).
  Log "End-of-day liquidation triggered".

- `split_buy(self, code, total_qty, price, hoga_gb) -> list[Order]`:
  Divide total_qty into split_count parts (33/33/34% per user decision).
  Submit first part immediately via order_manager.
  Return list of planned orders. Remaining parts need QTimer scheduling at split_interval_sec (45s) intervals.
  Note: Actual QTimer scheduling will be wired in main.py or via a simple _schedule_split method that uses QTimer.singleShot.
  For testability, split_buy returns the quantity splits and submits only the first. Callers handle timing.

- `split_sell(self, code, total_qty, price, hoga_gb) -> list[Order]`:
  Same as split_buy but for sell side.

- `_compute_splits(self, total_qty, count) -> list[int]`:
  Helper: divides total_qty into `count` parts. First (count-1) parts = total_qty // count, last part = remainder.
  E.g. 100 qty, 3 splits -> [33, 33, 34].

- `reset_daily(self)`:
  _daily_buy_blocked = False. Called at start of each trading day.

Write tests first (RED). Use mocks for order_manager, position_tracker, market_hours. Inject specific return values for each scenario. Then implement (GREEN).
