Create `kiwoom_trader/core/order_manager.py`:

Use PyQt5 import fallback pattern (try/except with MagicMock) matching session_manager.py pattern for cross-platform testing.

Class `OrderManager` (inherits QObject if PyQt5 available, otherwise object):
- Signals (pyqtSignal or MagicMock): `order_submitted(str)` (order_no), `order_filled(str, str, int, int)` (order_no, code, qty, price), `order_rejected(str, str)` (order_no, reason), `order_cancelled(str)` (order_no), `position_updated(str, int, int, int)` (code, qty, buy_price, current_price)
- `__init__(self, kiwoom_api, account_no: str)`: stores api, account_no, creates dict[str, Order] for _orders, screen counter from SCREEN.ORDER_BASE
- `submit_order(self, code, side, qty, price, hoga_gb) -> Order`: Creates Order dataclass, assigns screen_no (auto-increment), calls api.send_order(). On ret==0: transition to SUBMITTED, store order, emit order_submitted. On ret!=0: transition to REJECTED, emit order_rejected. Return order.
- `cancel_order(self, order_no) -> int`: Looks up order, determines cancel order_type (3 for buy, 4 for sell), calls api.send_order with org_order_no=order_no, price=0, hoga_gb="" for cancel.
- `handle_chejan_data(self, gubun, item_cnt, fid_list)`: Main event handler.
  - If gubun=="0" (order/execution notification):
    - Parse FIDs using api.get_chejan_data: ORDER_NO, CODE, ORDER_STATUS, ORDER_QTY, UNFILLED_QTY, EXEC_PRICE, EXEC_QTY, SELL_BUY
    - Apply strip() to all, abs(int(val or "0")) for numeric fields
    - CODE: remove leading "A" (code.replace("A", ""))
    - Determine new state from order_status + unfilled_qty:
      - If order not tracked (new order from outside): log warning and skip
      - If unfilled_qty == 0 and exec_qty > 0: FILLED
      - If unfilled_qty > 0 and exec_qty > 0: PARTIAL_FILL
      - If order_status contains "접수" or indicates acceptance: ACCEPTED
      - If order_status contains "취소": CANCELLED
      - If order_status contains "거부": REJECTED
    - Call `_transition_state(order, new_state)`
    - Update order.filled_qty, order.filled_price from exec data
    - Emit appropriate signal (order_filled, order_cancelled, order_rejected)
  - If gubun=="1" (balance notification):
    - Parse CODE, HOLDING_QTY, BUY_UNIT_PRICE, CURRENT_PRICE
    - Emit position_updated signal
- `_transition_state(self, order, new_state)`: Validates against VALID_TRANSITIONS. If illegal: log error and DO NOT transition. If legal: update order.state, log transition.
- `get_order(self, order_no) -> Order | None`
- `get_active_orders(self) -> list[Order]`: Returns orders not in terminal state.
- `_next_screen_no(self) -> str`: Auto-increment from SCREEN.ORDER_BASE, format as 4-digit string.

Write tests first using mock_kiwoom_api fixture. Mock get_chejan_data to return specific FID values for each test scenario.
