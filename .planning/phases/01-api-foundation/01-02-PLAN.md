    - KiwoomAPI.comm_rq_data(rq_name, tr_code, prev_next, screen_no) calls dynamicCall with CommRqData
    - KiwoomAPI.get_comm_data(tr_code, record_name, index, item_name) returns stripped string
    - KiwoomAPI.set_real_reg(screen_no, code_list, fid_list, real_type) calls dynamicCall with SetRealReg

  <name>Task 2: Create SessionManager with auto-reconnect and subscription restore</name>
  <files>
    kiwoom_trader/api/session_manager.py,

  </files>
  <behavior>
    - SessionManager._on_connect(0) resets retry count, starts heartbeat, emits session_restored
    - SessionManager._on_connect(non-zero) schedules reconnect
    - SessionManager._check_connection() when get_connect_state()==0 stops heartbeat, emits session_lost, schedules reconnect
    - SessionManager._schedule_reconnect() uses exponential backoff: BASE_DELAY * 2^retry_count, capped at MAX_DELAY
    - SessionManager._schedule_reconnect() stops after MAX_RETRIES (5)
    - SessionManager.track_real_subscription() stores subscription dict
    - SessionManager._restore_real_subscriptions() replays all tracked subscriptions via API
    - After reconnect success (_on_connect(0) with stored subscriptions), _restore_real_subscriptions is called
  </behavior>
  <action>
    **kiwoom_trader/api/session_manager.py**: Create SessionManager(QObject) per RESEARCH.md Pattern 4. Class attributes: MAX_RETRIES=5, BASE_DELAY_MS=5000, MAX_DELAY_MS=60000. Signals: session_restored(), session_lost(). Internal state: _retry_count, _reconnect_timer (QTimer singleShot), _heartbeat_timer (QTimer 30s interval), _real_subscriptions (list of dicts with screen_no, code_list, fid_list, real_type). Constructor takes kiwoom_api, connects to api.connected signal. Methods: start_monitoring(), _on_connect(err_code), _check_connection(), _schedule_reconnect(), _attempt_reconnect(), track_real_subscription(), _restore_real_subscriptions(). Use loguru for all state transitions (info for connect/disconnect, warning for reconnect attempts, critical for max retries reached).

    - test_reconnect_restores_subscriptions: _on_connect(0) with tracked subscriptions calls _restore_real_subscriptions
