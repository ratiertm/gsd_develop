  <behavior>
    TRRequestQueue:
    - enqueue(tr_code, rq_name, screen_no, inputs, prev_next, callback) adds request to deque
    - First enqueue when idle processes immediately and starts timer
    - Timer fires at interval_ms (default 4000) and processes next request
    - _process_next calls set_input_value for each input, then comm_rq_data
    - _process_next on empty queue stops timer and emits queue_empty
    - pending_count property returns queue length
    - Requests processed in FIFO order

    RealDataManager:
    - subscribe(code_list, fid_list, screen_no, real_type) calls api.set_real_reg and tracks subscription
    - unsubscribe(screen_no, code) calls api.set_real_remove
    - on_real_data(code, real_type, real_data) extracts FID values via api.get_comm_real_data, dispatches dict to subscribers
    - register_subscriber(real_type, callback) registers callback for data dispatch
    - _next_screen_no() auto-increments from SCREEN.REAL_BASE, returns 4-digit string
    - Tracks all subscriptions for SessionManager restoration
  </behavior>
  <action>
    **kiwoom_trader/api/tr_request_queue.py**: Create TRRequestQueue(QObject) per RESEARCH.md Pattern 2. Uses collections.deque. QTimer with configurable interval_ms (default 4000). Signal: queue_empty(). Methods: enqueue() adds dict to deque (tr_code, rq_name, screen_no, inputs dict, prev_next, callback), starts timer if not active and processes first immediately. _process_next() pops from deque, calls set_input_value for each key/value in inputs, then comm_rq_data. When deque empty, stops timer, emits queue_empty. Property: pending_count. Log every enqueue (debug) and dispatch (info) via loguru.

    **kiwoom_trader/api/real_data.py**: Create RealDataManager class. Constructor takes kiwoom_api and optional session_manager (for subscription tracking). Internal state: _subscribers dict (real_type -> list of callbacks), _subscriptions list (for restore), _screen_counter (starts at SCREEN.REAL_BASE). Methods:
    - subscribe(code_list, fid_list, screen_no=None, real_type="1"): If no screen_no, auto-generate via _next_screen_no(). Call api.set_real_reg(). Track subscription. If session_manager, call session_manager.track_real_subscription().
    - unsubscribe(screen_no, code): Call api.set_real_remove().
    - register_subscriber(real_type, callback): Add callback to _subscribers[real_type].
    - on_real_data(code, real_type, real_data): For each registered subscriber of real_type, call subscriber with (code, real_type). NOTE: FID extraction via get_comm_real_data should happen here inside the event context. Extract a standard set of FIDs (CURRENT_PRICE, VOLUME, EXEC_VOLUME, OPEN_PRICE, HIGH_PRICE, LOW_PRICE) into a dict, pass dict to subscribers.
    - _next_screen_no(): Increment _screen_counter, return as 4-digit string (format f"{counter:04d}").
