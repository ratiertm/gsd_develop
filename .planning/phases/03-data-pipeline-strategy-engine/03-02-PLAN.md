  <behavior>
    - StrategyManager loads strategies from config dict, creates indicator instances per strategy per stock
    - on_candle_complete(code, candle, indicators_dict) evaluates all enabled strategies for that code

    - Cooldown: after signal emitted for code+direction, ignore same code+direction signals for cooldown_sec seconds
    - Cooldown state resets daily (via reset_daily method)
    - execute_signal routes to either PaperTrader or RiskManager->OrderManager based on mode
    - Two presets: RSI reversal (RSI<30 buy, RSI>70 sell, priority=10), MA crossover (short EMA crosses above long EMA buy, crosses below sell, priority=20)
    - PaperTrader.execute_signal(signal) records virtual trade at signal.price
    - PaperTrader tracks virtual positions and computes P&L on sell
    - PaperTrader writes each trade to CSV (columns: timestamp, code, side, strategy, price, qty, amount, pnl, pnl_pct, balance, reason)
    - PaperTrader initial capital from config (default 10,000,000 KRW)
    - Default qty calculation: (capital * max_symbol_weight_pct / price) for paper trades
  </behavior>
  <action>
    1. Create strategy_manager.py:
       - StrategyManager(condition_engine, risk_manager, order_manager, config: dict)
       - config dict has "mode" ("paper"/"live"), "strategies" list, "watchlist" list
       - _load_strategies(config) parses strategy dicts into StrategyConfig objects with CompositeRule trees
       - on_candle_complete(code: str, candle: Candle, indicators: dict):
         a. Build context dict from indicators (indicator values + _prev values for cross detection)
         b. Add price/volume context: price=candle.close, volume=candle.volume
         c. Evaluate entry_rule and exit_rule for each enabled strategy assigned to this code
         d. Collect signals, resolve conflicts (highest priority per direction)
         e. Apply cooldown filter
         f. Route surviving signals to _execute_signal
       - _resolve_conflicts(signals: list[Signal]) -> list[Signal]: keep highest priority per (code, side) pair

       - reset_daily(): clear cooldown state
       - Two preset strategy configs built in _default_strategies():
         a. "RSI_REVERSAL": entry=AND(RSI < 30), exit=AND(RSI > 70), indicators={rsi: {period: 14}}, priority=10
         b. "MA_CROSSOVER": entry=AND(cross_above ema_short vs ema_long), exit=AND(cross_below ema_short vs ema_long), indicators={ema_short: {period: 5}, ema_long: {period: 20}}, priority=20
       - Store previous indicator values for cross detection
