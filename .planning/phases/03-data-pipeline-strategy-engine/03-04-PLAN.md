  <action>
Add state transition tracking to MarketHoursManager:

1. Add `_previous_state: MarketState | None = None` instance attribute in `__init__`.
2. Add `_state_callbacks: list[Callable[[MarketState, MarketState], None]] = []` instance attribute.
3. Add `register_state_callback(self, callback: Callable[[MarketState, MarketState], None]) -> None` method that appends to `_state_callbacks`.
4. Add `check_state_transition(self) -> tuple[MarketState, MarketState] | None` method that:
   - Calls `get_market_state()` to get current state
   - Compares against `_previous_state`
   - If different (and `_previous_state` is not None): updates `_previous_state`, fires all callbacks with `(old_state, new_state)`, returns `(old_state, new_state)`
   - If same: returns None
   - If `_previous_state` is None (first call): sets `_previous_state = current` and returns None (no transition on init)

This method is designed to be called from a QTimer in main.py. It does NOT change the existing polling interface — it adds to it.

Write tests FIRST in tests/test_market_hours.py (new TestStateTransition class), then implement.
